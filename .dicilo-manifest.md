# Dicilo Application Manifest

## Fixes & Patches
- 2026-04-07: Fixed Dashboard AlliesMap / Empresas de mi Interes logic. Businesses are now strictly filtered by user interests. Previously, "Cupones" mode bypassed the interests array, causing all users to see the same businesses regardless of their explicitly marked categories. It now strictly enforces `isMatchInterest || isFavorite` at all times.
- 2026-04-08: Replaced complex UI Popover with native `<select>` in `SuggestLocationDialog.tsx` & `OnboardingLock.tsx` to restore full touch capability on mobile devices.
- 2026-04-08: Integrated OpenStreetMap/LocationIQ geographical validation for user location suggestions, automatically bypassing the "Pending" state if data strongly correlates with global maps (`osm-validation.ts`).
- 2026-04-08: Fixed a silent UI bug in `QuickHighlightForm` where the local business search dropped valid entries because it strictly queried for `status == 'approved'`, forgetting that most businesses are saved under `status == 'active'`. Also bypassed the requirement of building a composite database index by moving the status filter to the browser/server memory.
- 2026-04-08: Created Google Gemini `moderateContentSkill` in `src/ai/moderation.ts` implementing a Zero-Shot toxic language detection system combined with a fast-path local dictionary filter. Integrated as a shield inside the `quick-highlight.ts` backend action to auto-reject abusive reviews in 12+ globally recognized languages.
- 2026-04-08: Fixed Firebase Role-Based Access Control (RBAC) desynchronization lock. Connected `MasterKeyDialog` and admin role selectors directly to `admin.auth().setCustomUserClaims()`, ensuring `firestore.rules` instantly grant valid users access without caching delays via forceful `revokeRefreshTokens`.
